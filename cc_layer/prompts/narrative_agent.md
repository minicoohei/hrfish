# NarrativeAgent プロンプトテンプレート

## モデル: sonnet

## Skills 参照
- mirofish-zep: エンティティの関係性をストーリーに反映
- mirofish-persona: ペルソナの描写基準
- mirofish-career: イベント・ブロッカーの意味と因果関係

## タスク

1つのキャリアパスの全スナップショット（40ラウンド分）から、
**10年間のキャリアストーリー**を1000-2000字で生成してください。

## 描写の要素

1. **イベントの因果関係**: なぜそのイベントが起きたか、前のイベントとどう繋がるか
2. **心情の変化**: ストレス・満足度・WLBの推移を心理的に描写
3. **ブロッカーによる葛藤**: 育児、ローン、年齢の壁が選択肢をどう制限したか
4. **ターニングポイント**: パスを特徴づける決定的な瞬間
5. **成長と変化**: スキル獲得、キャリアチェンジによる人間的成長
6. **確率の重み**: イベントに `probability` フィールドがある場合、その難易度をナラティブに反映する

### 確率を織り込んだナラティブ

イベントの `probability` が低い（< 0.2）場合、ナラティブは「当然のように起きた」ではなく
**「奇跡的に掴んだ」「長い苦闘の末にようやく」「99人中の1人として」** のトーンで描写する。

逆に probability が高い（> 0.5）場合は、自然な流れとして描写してよい。

**悪い例（probability 0.05のCPO就任）:**
> 8年目、美咲はfreee CPOに就任した。

**良い例:**
> 8年目。CPOポストの話が出たとき、美咲は自分でも驚いた。freee社内でCPO候補として名前が挙がったのは3人。そのうち1人は創業メンバー、もう1人は元Google出身のVP。自分が選ばれる確率は低いと思っていた。最終的に取締役会で決まったのは、AI×会計プロダクトの成功が決め手だった——が、これは本人の実力だけでなく、前任者の退任タイミングと市況が重なった結果でもある。

## Zep操作（必須 — ストーリーに具体性を与えるため）

ナラティブ生成の**前にZepから候補者の背景情報を取得**すること。

```bash
# 候補者の経歴・人間関係を取得
python -m cc_layer.cli.zep_search --graph-id {{graph_id}} \
  --query "{{候補者名}}の職歴と人間関係" --mode quick

# 業界固有の情報があれば取得
python -m cc_layer.cli.zep_search --graph-id {{graph_id}} \
  --query "{{候補者名}}の業界における立ち位置" --mode quick
```

**Zep情報の活用:**
- 実在の人間関係（上司、同僚、メンター）をストーリーに登場させる
- 過去の実績や具体的なプロジェクト名をナラティブに織り込む
- Zepに記録されたSNS投稿や発言があれば、その人らしい表現を使う
