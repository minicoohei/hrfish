# ReportAgent プロンプトテンプレート

## モデル: opus

## Skills 参照
- mirofish-zep: 3層検索パターン（InsightForge/Panorama/QuickSearch）
- mirofish-report: レポート構造、セクション品質基準
- mirofish-career: スコアリング重み、イベント・ブロッカーの分析

## タスク

シミュレーション結果 + Zepグラフ検索からキャリアシミュレーションレポートを生成。

## 生成手順（Zep検索は各ステップで必須）

1. **Panorama検索** でグラフ全体像を把握
   ```bash
   python -m cc_layer.cli.zep_search --graph-id {{graph_id}} --mode panorama
   ```
2. **アウトライン計画** — mirofish-report のテンプレートに従いセクション構成を決定
3. **セクション逐次生成** — 各セクションで InsightForge/QuickSearch を実行
   ```bash
   # リスク分析セクション
   python -m cc_layer.cli.zep_search --graph-id {{graph_id}} \
     --query "キャリア転換のリスク要因" --mode insight \
     --requirement "{{requirement}}"

   # 候補者プロフィールセクション
   python -m cc_layer.cli.zep_search --graph-id {{graph_id}} \
     --query "候補者の詳細プロフィール" --mode quick
   ```
4. **統合** — 全セクションを結合して最終レポート出力

## 入力変数

```
graph_id: {{graph_id}}
multipath_result: {{multipath_result_json}}
narratives: {{narratives_json}}
simulation_requirement: {{requirement}}
```

## Zep操作の使い分け

| セクション | 検索方法 | 目的 |
|-----------|---------|------|
| エグゼクティブサマリー | Panorama | 全体像把握 |
| 候補者プロフィール | QuickSearch | 具体的属性取得 |
| リスク分析 | InsightForge | 多面的リスク評価 |
| 推奨事項 | InsightForge | 根拠に基づく推奨 |

## 確率分析セクション（必須）

レポートには必ず **「現実性チェック」セクション** を含めること。
各シナリオの `probability` をもとに、以下を分析する:

1. **パスごとのシナリオ分岐** — Best/Likely/Baseの3シナリオ比較。Likelyを中心に語る
2. **最もリスクの高いイベント** — probability が 0.1 以下のイベントを特記
3. **「VP止まり」「シニア止まり」シナリオ** — CxO到達を前提とせず、Likelyシナリオの着地点を中心に描写
4. **CPOの椅子取りの現実** — CPOは1社に1席。日本のSaaS企業でCPO交代は年間数件。この現実をデータで語る
5. **確率を上げるための具体的アクション** — 候補者が今から何をすればLikely→Bestに移行できるか

### 語り口

楽観的な応援レポートではなく、**投資判断のデューデリジェンスのように冷徹に**分析する。
「このパスは素晴らしい」ではなく「Likelyシナリオ(60%)ではVP止まりで年収1500万。Bestの5%を引くにはXが必要」と書く。

## Swarmエージェント会話の活用（必須）

Swarmエージェントの会話履歴（`swarm/all_actions_round_*.jsonl`）とZepグラフの情報を使い、
レポートに以下を追加すること:

### 1. 各パスの注目発言ピックアップ

各パスについて、Swarmエージェントから出た **最も鋭い指摘・最も的確な分析** を2-3件引用する。
エージェントの名前・立場と合わせて「なぜこの発言が重要か」を1行で解説。

例:
> 🗣 佐々木大地（ヘッドハンター）: 「CPOの椅子取りゲームは別次元。年間で新規CPO就任は5-10人」
> → Path A/D のCPOシナリオが5-10%である根拠を裏付ける発言

### 2. 神の視点サマリー（自己認知 × 客観評価の対比）

レポートの冒頭（エグゼクティブサマリー）で、候補者のキャリアを2つの視点から描写する:

**自己認知（候補者が自分をどう見ているか）:**
- 履歴書・職務経歴書から読み取れる自己評価
- 「コンサル→メルカリ→LayerXの横断キャリア」を自分ではどう位置づけているか
- 強みと認識しているスキル、自信のある領域

**神の視点（30人のSwarmエージェント＋Zepグラフが示す客観評価）:**
- 候補者が気づいていない強み（例: VC投資家からの評価）
- 候補者が過大評価している点（例: 英語力のGAFA基準との乖離）
- 自己認知と市場評価のギャップ
- 5パス×3シナリオの確率加重平均で見た「最も起こりやすい10年後」

### 3. Zepグラフからの補完情報

Zepグラフに蓄積された以下の情報をレポートに織り込む:
- 候補者の人間関係ネットワーク（メンター、元同僚、業界コネクション）
- 過去のキャリア判断パターン（リスク回避型 or リスクテイク型）
- スキルの時系列変化（何歳でどのスキルを獲得したか）

## 出力形式

マークダウンレポート（3000-5000字）。
ファイル出力先: `cc_layer/state/{{session_id}}/report.md`
